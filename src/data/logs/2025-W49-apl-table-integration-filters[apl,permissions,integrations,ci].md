### Overview
Fixed two long-standing annoyances: dashboards crashed when an APL query called `table()`, and admins couldn't actually delete virtual fields because we were checking the wrong capability. Reshaped how integration filters load so the dynamic ones stop blocking the page, and stopped CI from running on release PRs since it had never caught anything and was just adding wait time.

### Highlights
- [#6373](https://github.com/axiomhq/app/pull/6373) — Taught `GetTables` in the APL parser to walk past function and variable `NameReference`s, skip `dataschema`, and extract literal names from `table()` calls — including resolving defaults from `declare query_parameters` via a new `getQueryParameterDefaults`. Exported `Utilities` and loosened the `toArray` signature for Bridge interop, with tests covering `table("name")`, variable-based `table(ds)`, and parameter defaults.
- [#6423](https://github.com/axiomhq/app/pull/6423) — Virtual-field deletion now checks `virtualFields:delete` instead of `datasets:update` via `usePermitStore().hasDatasetCapability(...)`, so admins can actually delete fields like the UI implies they should. Pulled the Manage/Delete/Close buttons into a `VirtualFieldPanelActions` component and reused it from `manageFieldAction`.
- [#6424](https://github.com/axiomhq/app/pull/6424) — Reshaped the integration filters API into `{ staticFilters, dynamicFilters }` with `IntegrationDynamicFilter` / `IntegrationStaticFilter` types. Built `integrationFiltersQuery` to merge them, `dynamicFilterQuery` for individual lookups, and a `useIntegrationFilters` hook with progressive 1h→2d loading, plus prefetching on org load so the UI doesn't sit empty on first paint. Killed the old integration filter / quick-query schema along the way.
- [#6425](https://github.com/axiomhq/app/pull/6425), [#6426](https://github.com/axiomhq/app/pull/6426) — Wrapped `loadMixpanel()` and `mixpanel.init(...)` in a try/catch so a failed dynamic import in `mixpanel-browser` stops throwing unhandled errors during boot. Also added a `check-release-pr` step to `ci-frontend.yml` that skips lint, typecheck, tests, coverage, and e2e for release PRs — CI has never caught anything on those, and the preview deploy still runs.
