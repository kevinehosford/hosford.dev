### Overview
Shipping week for the dashboards schema work: removed the view-model fields from the public contract, fixed the dirty-checking issue caused by premature payload wrangling, and started gating downloads on schema validation. Migrated the query type system across ~100 files to the 4-way discriminated union (`apl` / `mpl` / `structured` / `metrics-structured`) and finally added an MPL editor with language features. Wired up canary deploy aliasing so we can test non-main branches on the dev URL without blocking main.

### Highlights
- [#7360](https://github.com/axiomhq/app/pull/7360), [#7377](https://github.com/axiomhq/app/pull/7377), [#7378](https://github.com/axiomhq/app/pull/7378), [#7384](https://github.com/axiomhq/app/pull/7384), [#7403](https://github.com/axiomhq/app/pull/7403), [axiomhq/axiom#16550](https://github.com/axiomhq/axiom/pull/16550), [axiomhq/axiom#16611](https://github.com/axiomhq/axiom/pull/16611) — Dashboards payload cleanup: removed `datasetId` from query-chart and SmartFilter configs (view-only field, never part of the API), pulled the premature payload-wrangling serializer back out so in-memory state matches what was persisted (fixes dirty-checking), removed the wrong `isQueryChart` check that was breaking note rendering, and made download/copy run through `serializeDashboard` so exports are sanitized and validated against `dashboardSchema`. On the axiom side, dropped `datasetId` and `ChartDatasetId` from the public v2 spec and added missing `MetricsQueryOptions` + expanded filter ops.
- [#7389](https://github.com/axiomhq/app/pull/7389) — Migrated the console UI off ad-hoc type sniffing onto the existing 4-way discriminated union (`QueryForm` / `QueryRequest` / `QueryResponse` with `type: 'apl' | 'mpl' | 'structured' | 'metrics-structured'`) across ~100 files. Replaced raw string checks with typed guards (`isAplForm`, `isTextEditorForm`, `isBuilderForm`), killed property sniffing like `hasAplShape` / `.apl`-property access, and added the MPL editor with language features riding on the same shape.
- [#7381](https://github.com/axiomhq/app/pull/7381), [axiomhq/axiom#16606](https://github.com/axiomhq/axiom/pull/16606) — Multi-region resolution had an early return that was making everything fall back to the default region. Removed it so we actually use the dataset's region. Also added `useSecondaryYAxis` to the swagger so metrics queries don't lose the option when they leave dashboards.
- [#7419](https://github.com/axiomhq/app/pull/7419), [#7425](https://github.com/axiomhq/app/pull/7425) — Wired a `DEV_TARGET_BRANCH` repo variable and re-aliased the dev URL to the named branch's preview on `synchronize`, so canary branches can run on the shared dev URL without us merging anything to main. Added a `CanaryBanner` that shows up when `PUBLIC_DEV_TARGET_BRANCH` isn't `main`, with a clickable link to the branch on GitHub. Race condition's possible until we adjust Vercel's behavior; for now this is what we have.
