### Overview
Opened the RFC for the `features` paradigm refactor of `apps/console` — the goal is a documented architecture with lint backstops so the LLM-driven examples can point at real code (`features/<foo>`, `routes/<foo>`, `ui/primitives`). Cleaned up the tsconfig path aliases and chased the imports, drafted `CONTRIBUTING.md` from the agreed review guidelines, and added a pile of logging to chase a missing-logs support issue.

### Highlights
- [#6874](https://github.com/axiomhq/app/pull/6874) — RFC PR for `features/`: documented in `features/README.md`, `hubs/README.md`, `lib/README.md`, and `routes/README.md`, with a `scripts/extract/` folder for the ongoing refactoring tools. The point is "blessed" production examples we can hand to an LLM and trust.
- [#6949](https://github.com/axiomhq/app/pull/6949) — Cleaned tsconfig paths: removed the cruft and added `@/config`, `@/env`, `@/hubs`, `@/lib`, `@/features`. Then chased every `src/...` and deep-relative import across unit tests and source onto the aliases, with a lint rule to keep them that way.
- [#6976](https://github.com/axiomhq/app/pull/6976) — Got pulled into a support ticket where logs were missing. Found that tanstack start serializes server-fn errors in a way that bypasses the request logger, so surfaced more error info from `HTTPError` (trace-id, org-id) and unified the `HTTPError` detection to handle the serialized shape. Added a bunch of logging tests so we don't regress when this happens again.
- [#6977](https://github.com/axiomhq/app/pull/6977) — Endpoint form was portaled out of the tree containing `ActiveOrgProvider` and throwing a "missing required context" error. Wrapped it (and the rest of the places we have `OrgStore.Provider`) so the portal still resolves the active org.
- [#6945](https://github.com/axiomhq/app/pull/6945), [#6972](https://github.com/axiomhq/app/pull/6972) — Added `word-break: break-all` on the `pre` and `code > span` paths in the console's `MetadataTable` so users with long unspaced strings in event properties stop blowing out the row. And committed the team's review guidelines into `CONTRIBUTING.md` (author runs/validates/tests their changes; what reviewer approval signifies).
