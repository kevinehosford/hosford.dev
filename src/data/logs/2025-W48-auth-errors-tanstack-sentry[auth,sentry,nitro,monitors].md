### Overview
Big auth-error pass: invalid/missing access tokens now sign the user out cleanly with cookies and headers propagated, instead of leaving them stranded with a dead token. Replaced the legacy TrackJS/Sentry setup with the TanStack Start Sentry integration so error reporting goes through one Logger-shaped pipe. Also re-enabled chart zoom on monitor pages and stopped the nitro plugin from interfering with dev proxy encoding.

### Highlights
- [#6296](https://github.com/axiomhq/app/pull/6296), [#6323](https://github.com/axiomhq/app/pull/6323) — Made `catch-error` middleware async and taught it to recognize the known invalid-access-token error message: it calls `signOutViaAuthApi`, propagates `set-cookie` via `setResponseHeader`, deletes the user's access token, and logs structured context. Pulled the sign-out plumbing into `lib/auth/sign-out-utils.ts` (`signOutViaAuthApi`, `deleteAccessTokenForUser`, `extractSetCookieValues`), added `INVALID_TOKEN_ERROR` / `MISSING_ACCESS_TOKEN_ERROR` constants and `isMissingAccessTokenError`, and parallelized the user/org fetch while there.
- [#6346](https://github.com/axiomhq/app/pull/6346) — Swapped TrackJS and direct Sentry calls for the TanStack Start Sentry integration. Client and server both initialize through `src/lib/sentry-{client,server}.ts`, the Logger now forwards errors to Sentry with Vercel resource attributes and a `flush()`, and the error boundaries are slimmer because they don't talk to Sentry directly anymore. Wired Sentry into the Nitro plugin, the build, and CI version/env/CSP plumbing.
- [#6305](https://github.com/axiomhq/app/pull/6305), [#6307](https://github.com/axiomhq/app/pull/6307) — Better-auth IDP state was carrying a signature that the Axiom OAuth callback couldn't use, so added a `baseState` helper that strips everything after the `.` before forwarding. Then fixed the followup loop where a stray callback URL from a previous patch kept bouncing the user back to the same handler — now they actually land at `/`.
- [#6316](https://github.com/axiomhq/app/pull/6316) — Restored chart zoom on monitor pages: `RecentActivityAnomalyPanel` and `RecentActivityThresholdPanel` now pass an `onZoomTimeRange` handler to `QueryResults` that converts the `UPlotRange` to ISO via `epochToISO` using the user's timezone. Added a `MonitorViewPage` fixture and Playwright coverage so it doesn't regress quietly.
- [#6312](https://github.com/axiomhq/app/pull/6312), [#6298](https://github.com/axiomhq/app/pull/6298) — Gated `shouldBuildNitro` on `NODE_ENV === 'production'` so the nitro plugin stops mangling proxied request encoding during dev, and pruned the `mcp` workspace plus the trail of `@modelcontextprotocol/sdk` / Express 5 / body-parser entries out of the lockfile.
