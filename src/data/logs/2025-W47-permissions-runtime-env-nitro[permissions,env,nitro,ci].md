### Overview
Permission checks moved out of in-component 403 handling and into route loaders that throw `RoutePermissionError` for the boundary to catch — and then we mostly took them out of datasets/query/stream again because it turned out we didn't need to gate those at all. Also bumped Nitro to 3, taught the app to inject runtime env vars during SSR (so we have one access path), and added retry-with-backoff to every lazy import to stop dynamic import failures from blowing up the page.

### Highlights
- [#6207](https://github.com/axiomhq/app/pull/6207), [#6254](https://github.com/axiomhq/app/pull/6254) — Moved org-capability checks into `loader` functions on `/_authed/.../datasets|views|dashboards|monitors|...` so `AuthedErrorBoundary` can render `PermissionError` instead of the UI blocking on a permission probe. Prefetched datasets/stats while there to avoid first-render jank. Two days later, removed the checks again on `datasets`, `query`, and `stream` since the underlying capability ended up not actually gating those routes.
- [#6257](https://github.com/axiomhq/app/pull/6257) — Added runtime env injection: `__root.tsx` serializes the snapshot into `globalThis.__AXIOM_RUNTIME_ENV__` during SSR, `src/env/runtime` owns the read/write helpers, and `src/env/index.ts` reads from the injected snapshot (or `process.env` in tests). Added type-safe key utilities, auto-mapped `PUBLIC_` keys to shared keys, and shipped a custom `axiom-env/no-import-meta-env` ESLint rule to keep us from going back.
- [#6274](https://github.com/axiomhq/app/pull/6274) — Wrote `lazyWithDynamicImportRetry` (exponential backoff + cache-busting) and `isDynamicImportLoadError`, then migrated the lazy loads onto it. Error boundaries now reload the page on dynamic-import failures instead of tracking them as real errors — those usually mean a stale chunk after a deploy, not a real bug.
- [#6280](https://github.com/axiomhq/app/pull/6280) — Migrated from `nitropack` to `nitro@3` (alpha) and the new `nitro/vite` integration: switched imports to `nitro/*`, set `builder: 'rollup'`, swapped to `definePlugin`/`hooks.hook`, added a compatibility date, and dropped the esbuild target. Kept route rules/CSP/CORS untouched.
- [#6204](https://github.com/axiomhq/app/pull/6204), [#6226](https://github.com/axiomhq/app/pull/6226), [#6249](https://github.com/axiomhq/app/pull/6249), [#6228](https://github.com/axiomhq/app/pull/6228), [#6232](https://github.com/axiomhq/app/pull/6232) — Dashboards e2e now routes API calls through Playwright's request context with explicit polling/assertions; removed the duplicate `OrgModal` from `PrimaryNavBar` and added a test for the new-org modal; added Play UI smoke tests across Datasets/Query/Stream/Dashboards behind a dedicated project config; expanded the region utilities with `getEdgeDeploymentLabel`/`isEuEdgeRegion`/`US_EAST_REGION_ID`; and reverted the uplot bump back to 1.6.24 after the tooltip regression.
