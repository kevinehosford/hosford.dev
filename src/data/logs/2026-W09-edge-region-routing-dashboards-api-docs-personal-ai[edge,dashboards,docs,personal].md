### Overview
Edge routing now actually picks the base URL from the target dataset's region instead of the env default. Drove the public Dashboards API forward across three repos (Console schema sync, axiom-side spec sanitization, docs site spec + endpoint pages), added lint rules for the new `module/` boundary, and wrote the onboarding banner so it actually clears when you finish setup.

### Highlights
- [#7284](https://github.com/axiomhq/app/pull/7284) — Console edge querying now resolves the edge base URL using the target dataset's `region` (when available) instead of always using the environment default, with a fallback when parsing/resolution can't land a region. New cache-key semantics and tests included.
- [#7197](https://github.com/axiomhq/app/pull/7197) followups: [axiomhq/axiom#16505](https://github.com/axiomhq/axiom/pull/16505), [axiomhq/axiom#16525](https://github.com/axiomhq/axiom/pull/16525), [axiomhq/docs#570](https://github.com/axiomhq/docs/pull/570) — Wired the dashboards schema work into the v2 dashboards integration on the axiom side, did a pass to sanitize the public-facing schema so consumers only see the parts they should use, and shipped the v2 dashboards spec + five endpoint pages (list/get/create/update/delete) into docs.json so the Dashboards API has an actual reference.
- [#7325](https://github.com/axiomhq/app/pull/7325), [#7340](https://github.com/axiomhq/app/pull/7340), [#7314](https://github.com/axiomhq/app/pull/7314) — Added lint rules for `.../module/*` so the new intermediary space enforces boundary guidelines ("don't use react context directly", "wrap react query options factories in a hook"), with one controller extracted as an example and a test scaffold. Documented the high-level console/nexus topology in `apps/console/docs/axiom-architecture.md` (Mermaid diagram of Console↔Nexus/Apex/Edge and Nexus↔Core/Atlas), and added a long-form `2026Q1-axiom-metrics-preview.md` audit of the current Metrics DB preview implementation as the basis for the next refactor.
- [#7343](https://github.com/axiomhq/app/pull/7343), [#7296](https://github.com/axiomhq/app/pull/7296) — Onboarding banner wasn't going away after setup because the `send-data` complete step was never called. Encapsulated the react-query calls into hooks and extracted a controller hook for the submission so the step fires properly. Also moved the `RegionSelector` from the legacy `UsagePage` to `UsagePageNext` so non-EU users (i.e. nearly everyone) can actually filter usage by region.
- [kevinehosford/personal-ai#30](https://github.com/kevinehosford/personal-ai/pull/30), [kevinehosford/personal-ai#31](https://github.com/kevinehosford/personal-ai/pull/31) — On the personal side: added auto-deploying Fly.io PR previews (ephemeral app per PR, sticky comment with URL, auto-destroy on close) with the bot degrading gracefully when `TELEGRAM_BOT_TOKEN` is missing, then built a cron-style scheduler (60s tick, ET tz, state-file dedup) plus MorningBrief and EveningReflection skills wired up at 8:30 AM / 4:30 PM ET.
