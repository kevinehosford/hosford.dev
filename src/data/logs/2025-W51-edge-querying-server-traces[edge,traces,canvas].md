### Overview
Shipped feature-flagged edge querying — APL and dataset queries can route through the edge API with an automatic dashApi fallback when params don't resolve. Restored server traces (broken since we moved off NodeSDK), and turned off the `query-canvas` feature flag so canvas is just always on now.

### Highlights
- [#6589](https://github.com/axiomhq/app/pull/6589) — Wired the `fe-edge-querying` flag through the Console query flows. New `queryAplReact` / `queryAplNonReact` router entry points, `aplQueryOptions` / `eventTimelineQuery` / `spotlightFormQuery` / integration-filter / traces/logs/virtual-field-preview factories now take `queryClient` + `edgeEnabled`, plus `edgeOptions` and `resolveEdgeParams` helpers and a separate edge cache/query key. Added `createDefaultFetcher` for `atlas-api` (with org header injection), extended logging with `EDGE_QUERY`, and added optional AXIOM feedback env vars.
- [#6561](https://github.com/axiomhq/app/pull/6561) — Server traces had been silently empty since we swapped tracing stacks; replaced `NodeSDK` with `NodeTracerProvider` + `registerInstrumentations`, `AlwaysOnSampler`, and composite W3C + optional Sentry propagators, then re-wired logging/middlewares and shutdown so trace IDs actually emit reliably. Spans show up in nexus traces again.
- [#6599](https://github.com/axiomhq/app/pull/6599) — Removed the `query-canvas` feature flag entirely. `runQueryToCanvas`, the `Cmd+Shift+Enter` shortcut, chart sync, and refresh are unconditional in `QueryForm` and `QueryForm-canary`; `shouldSkipQueryForCanvas` lost its `hasCanvasFlag` param; `RunQueryButton` always supports Shift+Enter to add to canvas for APL; and `useQueryLaunchPadStore` decides the canvas tab purely on `hasChartsInCanvas`.
- [#6596](https://github.com/axiomhq/app/pull/6596), [#6588](https://github.com/axiomhq/app/pull/6588) — Stopped rendering the icon/label inside `ChartHeader` when `headerText` is already set, so metric/stat charts don't show duplicated titles. Also exempted the Integrations page from the "Apps are not available" alert when running in dev so devs in EU/AP-style limited regions can still work locally.
