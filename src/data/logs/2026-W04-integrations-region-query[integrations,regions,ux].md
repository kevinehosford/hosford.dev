### Overview
Another pass on integrations gating, this time using an actual `useQuery(getAxiomRegionsQuery)` so we compare the org's `defaultRegion` against the environment default instead of guessing. Documented the deprecated dropdown components so people stop reaching for them, and made chart-save failures surface real feedback instead of silently dying.

### Highlights
- [#6826](https://github.com/axiomhq/app/pull/6826) — Integrations page now fetches regions via `useQuery(getAxiomRegionsQuery)` and gates access by comparing `environmentDefault` to the lowercased org `defaultRegion`. Renders a centered `LoadingIndicator` during fetch and a danger `Alert` on failure, then shows the legacy-EU info banner when the org's default region differs from the environment default and `edgeIntegrationsEnabled` is false. EU edge no longer trips the banner.
- [#6844](https://github.com/axiomhq/app/pull/6844) — Chart save was failing silently. Now we surface `Notification.error` with rule messages when APL/builder validation fails, show another notification on schema parse failure and log via `Logger.error(LogService.GENERAL, ...)`, and added a form-level `handleSubmit` error handler to extract nested validation messages. Still don't know why some of these fail, but at least users will see something.
- [#6828](https://github.com/axiomhq/app/pull/6828) — Docs-only: updated `DropdownMenu_DEPRECATED.tsx` and `Dropdown_DEPRECATED.tsx` JSDoc to point at `primitives/Menu.tsx` (`MenuProvider` / `MenuButton` / `Menu` / `MenuItem`) and `primitives/Popover.tsx` for overlays. We've been migrating off `rc-dropdown` for a while; this is the breadcrumb so new code stops reintroducing it.
