### Overview
Extended client-side observability with `session_id`, `route_id`, and `pathname` so we can group user behavior per tab/route. Migrated the Playwright suite to staging so it stops fighting the preview env, drafted a zod schema for dashboard payloads (shadow-validating reads/writes for now), and fixed the time-bin resolution that was getting dropped when switching between APL and the builder.

### Highlights
- [#7101](https://github.com/axiomhq/app/pull/7101) — Extended `ObservabilityContext` with `session_id` (per-tab `crypto.randomUUID()`), `route_id`, and `pathname`. Logs and Sentry both pick it up — `session_id` groups everything happening in one tab, `route_id` lets us aggregate behaviors per route without dealing with `pathname` cardinality.
- [#7105](https://github.com/axiomhq/app/pull/7105) — Pointed Playwright at staging: `ATLAS_API_URL`, `LOGIN_URL`, and the apex URLs all swap to staging, tests use `App.urls.build` instead of hardcoded org ids, HAR files updated to the staging org, and the unstable ones got `fixme`'d so we can land the migration without unrelated noise.
- [#7119](https://github.com/axiomhq/app/pull/7119) — Drafted a zod schema for dashboard payloads (we need a public spec for external users to author dashboards). Added shadow validation on GET/PUT of individual dashboards plus POST to the index — just logging mismatches for now so we can iterate the schema against real traffic without breaking anyone.
- [#7145](https://github.com/axiomhq/app/pull/7145) — Switching between APL and the builder was dropping the time-bin resolution, which meant some dashboards came back with nothing in the datasets field. Now if the APL contains `bin(_time` or `bin_auto(_time`, we always set resolution to `auto` on the way back to structured.
- [#7114](https://github.com/axiomhq/app/pull/7114) — Endpoints page was being hidden in EU prod because of the region-limited check, but legacy EU is supposed to still see them. Added `isEnvEuProd()` to the gating condition so they're back.
