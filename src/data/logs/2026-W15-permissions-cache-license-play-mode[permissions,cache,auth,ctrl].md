### Overview
Fixed a real UX problem where settings pages were 404-ing on permission failures (so users had no recourse but to ping support), tightened a pile of dashboard/query bugs that crept in with recent caching work, and hardened play-mode authorization. Bumped Turbo and downgraded TanStack Start to dodge the blank-screen issue. Kept the `ctrl` scaffolding moving on the side.

### Highlights
- [#7626](https://github.com/axiomhq/app/pull/7626) — Settings routes were calling `notFound()` when a user lacked the capability for the resource. Replaced those with `throw new RoutePermissionError(...)` and nested the permission-protected pages under `settings/_authed` so a shared `errorComponent` can render the real "you don't have permission" message with actionable detail.
- [#7637](https://github.com/axiomhq/app/pull/7637) — The "ensure latest version" hook was reading from a static file (artifact of a previous reshuffle) instead of the backend `package.json`. Pointed it at the right source so clients actually get notified after deploys. Also moved reusable app-level components up to `src/*` so other surfaces can grab them.
- [#7652](https://github.com/axiomhq/app/pull/7652), [#7661](https://github.com/axiomhq/app/pull/7661) — Stopped persisting `getOrgQuery` and `orgLicenseQuery` to IndexedDB because org license info was getting stuck behind the cache (visible bug: license changes didn't take effect). And caught the blank-screen failures by making `ErrorBoundary.getDerivedStateFromError` accept `unknown` and always coerce to an `Error`, plus rolled `@tanstack/react-start`/`react-router` back to the last stable revs since the latest were the source of the blank screens.
- [#7659](https://github.com/axiomhq/app/pull/7659) — Auth hardening for play mode: added `PLAY_MODE_ORG_ID` to the envs that support it, and we now reject requests whose `orgId` doesn't match. Also use the HTTP status from `checkAPITokenWithAtlas` (via its `errorStatus`) instead of always returning a generic 401.
- [#7628](https://github.com/axiomhq/app/pull/7628), [#7638](https://github.com/axiomhq/app/pull/7638), [#7639](https://github.com/axiomhq/app/pull/7639) — Moved the dashboards search-param schema from `dashboards/index` up to `dashboards/route` so visiting `dashboards/new` doesn't trip a tanstack invariant on the missing params. Fixed a hydration-mismatch error on `/query` (React error #418). Bumped Turbo from 2.6.1 → 2.9.4 with the codemod.
- [axiomhq/ctrl#12](https://github.com/axiomhq/ctrl/pull/12), [axiomhq/ctrl#13](https://github.com/axiomhq/ctrl/pull/13), [axiomhq/ctrl#23](https://github.com/axiomhq/ctrl/pull/23) — On the `ctrl` repo: renamed the microfrontends and packages, adjusted the mfe config to include `www`, bumped Turbo, and wired up the Vercel toolbar plus the first round of mfe-development docs.
