### Overview
Fixed two MPL-monitor bugs (anomaly create button did nothing, pie/stat couldn't reduce mpl results), bumped the mpl package, and pulled the npm token out of `.npmrc` now that the packages are public. Started scaffolding a new `ctrl` monorepo with microfrontends.

### Highlights
- [#7597](https://github.com/axiomhq/app/pull/7597) — Anomaly monitor create button did nothing for MPL because the create path had bespoke branching that didn't know what to do with the MPL shape. Replaced it with `toTypedQuery` (already used in `updateMonitor()`) so APL / MPL / structured all go through the same conversion + timing.
- [#7618](https://github.com/axiomhq/app/pull/7618) — Pie and stat charts on MPL data weren't reducing properly because they depend on `queryMeta` for the scalar reduction, and we only computed `queryMeta` ahead of time for `metrics-structured`. Added MPL parsing + transformation extraction so the same metadata is available. Not the cleanest, but it works.
- [#7598](https://github.com/axiomhq/app/pull/7598), [#7589](https://github.com/axiomhq/app/pull/7589) — Bumped `@axiomhq/mpl` and dropped the vendored CodeMirror MPL integration from `@axiomhq/editor` (removed the `./mpl/codemirror` export and its entrypoint), and removed the pnpm patch on `@axiomhq/mpl` while there. Pulled the `//registry.npmjs.org/:_authToken=${NPM_AUTOMATION_TOKEN}` line out of `.npmrc` since the packages are public now.
- [axiomhq/ctrl#1](https://github.com/axiomhq/ctrl/pull/1) — Scaffolded a new monorepo with microfrontends: 4 apps (`console`, `login`, `api`, `www`), TanStack Start/React + Astro + Hono + Drizzle + better-auth, Vercel microfrontends with dynamic port assignment, pnpm workspaces with catalog deps, a shared UI package, and Sentry instrumentation.
