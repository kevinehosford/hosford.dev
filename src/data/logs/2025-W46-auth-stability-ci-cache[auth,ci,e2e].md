### Overview
Chased a pile of auth weirdness: 403s were signing people out, sign-out wasn't fully invalidating the atlas token, and the cookie cache was sitting on the wrong config so every request was paying for it. Re-enabled the E2E job as a required check after stabilizing a few more suites, and switched our pnpm caching over to the official setup-node approach.

### Highlights
- [#6088](https://github.com/axiomhq/app/pull/6088), [#6143](https://github.com/axiomhq/app/pull/6143) — Stopped auto-signing people out when `dashApi.getOrgs()` returns a 403 (commented out the `signOutHandler()` in `SSEStoreBase`/`StoreBase`), and made `getSavedActiveOrgId` validate the cached org id against actually-accessible orgs before reusing it. Then fixed the inverse problem: sign-out was deleting the atlas token from the account row but leaving the row in place, so subsequent logins reached for an invalid token. Now we only invalidate + delete the device session, added a self-service "log out" link that also nukes the account row when you actually need a reset, and re-enabled the sign-out e2e test that confirms the redirect to the login page.
- [#6097](https://github.com/axiomhq/app/pull/6097) — Moved `expiresIn` and `cookieCache` under a `session` block in the better-auth config, disabled `disableSessionRefresh`, and dropped the cookie lifetime to 29 days. Sole goal was knocking ~100ms off every request, which it appears to have done.
- [#6086](https://github.com/axiomhq/app/pull/6086), [#6084](https://github.com/axiomhq/app/pull/6084) — Replaced the global play-mode flag with `isPlayMode(request)` inside `get-access-token-context` so play and real org requests don't share state, and reordered the `get-user-status` server function so `atlasApiMiddleware` runs before `ensureSession` (turns out order matters when one depends on the other being set up).
- [#6082](https://github.com/axiomhq/app/pull/6082), [#6127](https://github.com/axiomhq/app/pull/6127), [#6130](https://github.com/axiomhq/app/pull/6130) — Stabilized the Playwright suite enough to flip `E2E Tests` back on as a required check on Axiom-owned PRs: added a dashboards setup/teardown that seeds via `emptyDashboard`/`dashboardWithCharts` fixtures, refactored the notifier cleanup to actually clean notifiers, and switched our pnpm cache to use `pnpm/action-setup` + `actions/setup-node` with `cache: 'pnpm'` per the official docs. While there, expanded cache paths to cover `node_modules` in repo root, `frontend-server`, `frontend-swagger`, and `mcp`.
