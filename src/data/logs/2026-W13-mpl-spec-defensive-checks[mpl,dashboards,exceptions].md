### Overview
Cleanup week chasing exceptions: dashboards now tolerate missing `queryOptions` everywhere, the APL/MPL empty-check stops blowing up on `undefined`, and the dashboard form survives transitory null states during transitions. Documented MPL natively in the Dashboards v2 schema, swapped the "regions" copy for "edge deployments", and republished mpl/mpl-codemirror packages.

### Highlights
- [axiomhq/axiom#16818](https://github.com/axiomhq/axiom/pull/16818) — Added native MPL to the Dashboards v2 OpenAPI spec: new `*NativeMplQuery` schemas plus an `MplString` definition, threaded through every chart type that can accept a query. Generated specs (`dashboards-openapi.json` and `dashboards-openapi-full.json`) updated.
- [#7533](https://github.com/axiomhq/app/pull/7533) — During the OAuth callback, user status comes back as an empty object initially. The server-fn return type didn't catch the empty case so we were persisting `{}` to the cache and tripping login errors. Also it shouldn't have been persisted in the first place since it represents your latest org access. Removed it from the persist list and added cache-clear to the auth-error handlers.
- [#7555](https://github.com/axiomhq/app/pull/7555), [#7559](https://github.com/axiomhq/app/pull/7559), [#7563](https://github.com/axiomhq/app/pull/7563), [#7567](https://github.com/axiomhq/app/pull/7567) — Made `queryOptions` optional throughout the console query pipeline so older/external payloads stop tripping validation; added defensive checks in `isEmptyOrPlaceholderApl` for the undefined APL/MPL paths seen in Sentry; added defensive checks against the transient null states that form transitions can produce (working on tightening the types properly too); and made `getQueryMetaFromResult` fall back to client-provided metadata when `result.tables` is null/empty (legacy/usage page responses).
- [#7535](https://github.com/axiomhq/app/pull/7535), [axiomhq/mpl#2006](https://github.com/axiomhq/mpl/pull/2006) — Renamed "Region" to "Edge deployment" across dashboards, datasets/views, and monitors (table headers and sidebar filter section titles only). Switched the mpl/mpl-codemirror release flow to publish to GitHub Packages via `actions/setup-node` with `registry-url`/`scope` and `GITHUB_TOKEN`, replacing the prior `JS-DevTools/npm-publish` + npmjs token setup.
