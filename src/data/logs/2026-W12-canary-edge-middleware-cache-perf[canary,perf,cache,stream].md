### Overview
Two big features: opt-in canary routing via an edge-middleware cookie so users can preview specific deployments, and React Query persistence to IndexedDB so refreshes show data immediately while we revalidate in the background. Also a perf pass on dashboards and the stream.

### Highlights
- [#7497](https://github.com/axiomhq/app/pull/7497), [#7500](https://github.com/axiomhq/app/pull/7500) — Added an experimental-features toggle in settings that sets an `x-axiom-console-canary: 'enable'` cookie when enabled. Edge middleware reads the cookie and rewrites to the specific preview deployment. Then broadened `config.matcher` from exclusion-based to a catch-all `/:path*` because assets needed it too.
- [#7525](https://github.com/axiomhq/app/pull/7525) — Persisted the React Query client to IndexedDB so a page refresh shows whatever was there, then fetches in the background according to existing `staleTime`s. Busts on version change, sign-out, and org change. Cached queries: `appViewsQuery`, `databaseStatsQuery`, `datasetInfoQuery`, `getAnnotationsQuery`, `getAxiomRegionsQuery`, `getDashboardsQuery`, etc. Removed the localStorage dataset cache while there since this supersedes it.
- [#7518](https://github.com/axiomhq/app/pull/7518) — Virtualized the dashboards list with `react-virtuoso`, tightened dirty-state comparison so it stops reporting clean dashboards as dirty, and improved search latency by deferring the filter.
- [#7524](https://github.com/axiomhq/app/pull/7524) — Stream's "scroll to get more rows to load" issue is probably just it being frozen on render, so cut down on observables, made function calls lazy, and made several components lazy too. Touches the table render path and context menu; row updates and field typing should be unchanged.
